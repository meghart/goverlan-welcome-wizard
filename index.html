Team Collaboration

To activate team collaboration, you need to configure a common database back end with your teammates. Database configurations can be published to your team in multiple ways depending on your needs.

In large environments, a GPO or a Goverlan Central Server is appropriate. However you can also push these settings manually for a simpler deployment.

Ok, let’s use a GPO to configure Goverlan!

You will find a Goverlan Group Policy Admin Template in this directory. This GPO templates can be used to configure the behavior and settings of Goverlan both on the operator and client sides.

Publish this GPO Admin Template to your Active Directory, then use the Group Policy Management snap-in to configure Goverlan’s database back-end across your network.

Team Collaboration

Ok, let’s use the Goverlan Central Server to configure Goverlan!

The Goverlan Central Server (GCS) is used when secure and instantaneous configuration is needed. It is also used to enable centralized auditing of Goverlan events.

1. Download and install the GCS on a Windows Server
2. Configure the settings & auditing you need
3. Publish the GCS to your infrastructure via DNS or GPO

Detecting User's Machines

Then we need a Goverlan Central Data Repository (GCDR) before this feature can work!. A GCDR is a simple network share that is accessible from your client machines, and modifiable by Domain Users. Don’t worry, only lightweight and non-confidential data will be stored there!

If this is the first time that a GCDR is being configured, then create a network share on a server, allow read/write access to Domain Users and enter the network share location here:
